# Devotional Question Architect

This skill serves two modes:

1. `Front-end architect mode`
   - define the controlling questions before the deep work starts
2. `Back-end discussion mode`
   - generate discussion and reflection questions after the analysis is already done

Do not confuse these two modes.

## Mode 1: Front-end architect

Use this mode when the user gives:

- a Bible passage
- a chapter range
- a sermon topic
- a spiritual burden

and the real need is:

- what question should drive the study
- what sequence of questions should structure the study
- what each later stage should try to answer

### Core task

Translate a passage into a clean study spine:

1. `Primary interpretive question`
   - what is the author really trying to answer here
2. `Exegesis questions`
   - what do we need to observe in the text
3. `Biblical theology questions`
   - what does this reveal about God's covenant, kingdom, presence, or redemptive movement
4. `Formation questions`
   - what human condition is exposed
   - what kind of response is God shaping

### Architect rules

- Start with one controlling question, not five equal questions.
- Make the first question passage-shaped, not generic.
- Keep each downstream question stage-specific.
- Do not ask application questions before interpretive questions.
- Do not flatten every passage into "how should we obey".
- If the passage is narrative, let conflict, turning point, and narrator emphasis shape the questions.
- If the passage is discourse, let argument flow and repeated logic shape the questions.
- If the passage is list-heavy or administrative, ask why the author devoted space to it.

### Architect output contract

Use these headings:

- `Controlling Question`
- `Exegesis Questions`
- `Biblical Theology Questions`
- `Spiritual Formation Questions`
- `Recommended Study Flow`

## Mode 2: Back-end discussion mode

## Question ladder

Prefer this progression:

1. observation
2. theological significance
3. heart exposure
4. faithful response

## Question rules

- Write 2 to 4 discussion questions unless the user asks for more.
- Avoid yes/no questions.
- Avoid vague "share anything" prompts.
- Make each question answerable from the actual passage movement.
- At least one question should expose desire, fear, or trust.
- At least one question should move toward Christ or gospel hope.
- Keep discussion questions clearly different from the architect-mode questions.
- Discussion questions should feel answerable in community, not like private research tasks.

## Golden sentence rules

When asked for a closing line:

- write one sentence
- keep it memorable, pastoral, and passage-rooted
- avoid slogans detached from the text

## Output contract

For discussion mode, use these headings:

- `Discussion Questions`
- `Reflection Prompt`
- `Golden Sentence`
